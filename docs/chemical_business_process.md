# 化学品出入库与危化品管控业务流程

## 角色与权限

- 系统管理员：admin，可维护账号、化学品、库存、审批和审计。
- 库管员：keeper，可维护化学品台账、人员卡片、执行入库、出库、归还、盘点。
- 安全管理员：safety_manager，可审批危险化学品领用，可查看审计。
- 普通使用人：operator，可刷卡领用、提交危化品申请、登记实际用量。

## 化学品建档

1. 库管员录入化学品编号、名称、别名、CAS、规格、库存单位、存放位置、所属项目。
2. 系统要求录入危险性、危险等级、是否危化品、管控级别和储存要求。
3. 瓶身照片可先走 OCR，识别化学品名称、CAS、规格和危险性后再保存到台账。
4. 危险等级大于等于 3、标记为危化品、或管控级别不是 normal 的化学品进入危化品管控视图。

## 入库流程

1. 库管员选择已有化学品，或先新增化学品台账。
2. 填写入库数量、项目、备注。
3. 后端写入 inventory_transactions 入库流水。
4. 系统增加 chemicals.stock，并在 dashboard 中刷新库存和低库存预警。

## 普通化学品出库流程

1. 使用人刷卡或手动录入卡号。
2. 系统校验卡片是否已注册、是否绑定人员、是否处于 active 状态。
3. 选择化学品、填写领用数量、用途和项目。
4. 普通化学品库存充足时，系统直接生成出库流水并扣减库存。
5. 如结合天平称重，可额外保存 usage_records，记录使用前重量、使用后重量和实际用量。

## 危险化学品出库流程

1. 使用人刷卡核验身份。
2. 选择危化品并填写数量、用途、项目和申请原因。
3. 普通用户提交后，系统自动生成 approval_requests，库存暂不扣减。
4. 安全管理员或系统管理员在审批中心审核。
5. 审批通过后，系统自动生成出库流水并扣减库存。
6. 审批驳回时，仅保留审批记录和审计日志，不改变库存。

## 归还流程

1. 使用人或库管员选择化学品并填写归还数量。
2. 系统生成 return 流水。
3. 系统增加库存，并保留归还人员、卡号、项目和备注。

## 盘点调整流程

1. 库管员或管理员根据盘点结果选择化学品。
2. 填写调整后的真实库存。
3. 系统生成 adjust 流水，记录调整前和调整后库存。
4. 调整动作进入 audit_logs，便于追溯。

## 危化品管控要点

- 身份管控：出库前必须刷卡或登录，卡片需绑定人员。
- 审批管控：危险等级高、危化品标记或特殊管控级别自动触发审批。
- 库存管控：库存不足时拒绝出库；低于库存下限时进入预警。
- 储存管控：台账记录储存要求和位置，方便现场检查。
- 留痕管控：登录、OCR、建档、出入库、审批和盘点均写入审计日志。
- 称重管控：领用前后称重形成实际用量记录，和库存流水互相校验。
